## Implementation Steps

### Step 1 — A11Y-01: Hamburger button aria-label (state-aware)

File: `next-app/components/Header.tsx`

**Current state:** The button has `aria-label="Open menu"` — static and correct for the closed state. However, when the menu is open, the label should read "Close menu" to accurately reflect the action.

**Fix:** Make the `aria-label` dynamic and add `aria-expanded` to communicate state to assistive technology.

```tsx
<button
  className="md:hidden border border-[#e2e8f0] bg-transparent text-[#1f3a5f] text-[1.2rem] px-2.5 py-1.5 rounded-[10px]"
  aria-label={menuOpen ? "Close menu" : "Open menu"}
  aria-expanded={menuOpen}
  aria-controls="mobile-nav"
  onClick={() => setMenuOpen((prev) => !prev)}
>
  &#9776;
</button>
```

Also add `id="mobile-nav"` to the `<nav>` element so `aria-controls` has a valid target:

```tsx
<nav
  id="mobile-nav"
  className={`items-center gap-2 ${
    menuOpen
      ? "flex flex-wrap w-full pt-2 pb-3 border-t border-[#e2e8f0] bg-white"
      : "hidden md:flex"
  }`}
  aria-label="Main navigation"
>
```

### Step 2 — A11Y-02: Team card focus states

File: `next-app/app/team/TeamGrid.tsx`

**Current state:** The LinkedIn overlay `<a>` becomes visible on `group-hover` and `group-focus-within`, but the link itself has no visible focus ring. A keyboard user tabbing to the link gets no visual indicator.

**Fix:** Add `focus-visible:ring-2 focus-visible:ring-white focus-visible:ring-offset-2 focus-visible:ring-offset-[#113b78]` to the overlay anchor. Also add `focus-visible:opacity-100` so the overlay is visible when the link receives focus directly.

Replace the overlay anchor's `className`:

```tsx
<a
  className="absolute inset-0 grid place-items-center bg-[rgba(17,59,120,0.72)] opacity-0 transition-opacity duration-[220ms] group-hover:opacity-100 group-focus-within:opacity-100 focus-visible:opacity-100 focus-visible:outline-none focus-visible:ring-4 focus-visible:ring-white focus-visible:ring-inset"
  href={member.linkedin}
  target="_blank"
  rel="noreferrer"
  aria-label={`${member.name} on LinkedIn`}
>
```

The `focus-visible:ring-inset` approach ensures the ring renders inside the overlay boundary without layout shift. `focus-visible:outline-none` suppresses the browser default outline so the ring is the only indicator.

### Step 3 — A11Y-03: Hero overlay contrast check

File: `next-app/app/page.tsx`

**Analysis:**

The hero text renders white (`text-white` / `text-white/90`) over the gradient overlay:
- Start: `rgba(28,167,166,0.82)` — teal-based
- End: `rgba(31,58,95,0.88)` — navy-based

The text sits in the left portion of the section, which maps to the teal side of the gradient at approximately `rgba(28,167,166,0.82)`.

WCAG AA requires a minimum contrast ratio of 4.5:1 for normal text and 3:1 for large text (18pt / 14pt bold). The h1 is `clamp(2.2rem, 5vw, 3.6rem)` bold — clearly large text.

White (`#ffffff`) against `rgba(28,167,166,0.82)` blended over a video produces an estimated contrast of ~4.2:1 at the lightest edge. The `text-white/90` subheading drops slightly lower at ~3.8:1.

**Result:** The h1 passes AA for large text (3:1 threshold). The body paragraph at `text-white/90` with font-size `1.15rem` is borderline for normal text at the teal side.

**Fix:** Increase the gradient opacity floor slightly at the start to deepen the teal overlay and ensure all text elements clear AA.

Change the overlay gradient from:
```
rgba(28,167,166,0.82) 0%
```
to:
```
rgba(20,120,120,0.88) 0%
```

This darkens the teal channel while keeping the visual character, pushing the contrast ratio for white text to ~5.2:1 at the lightest point — comfortably above AA for both large and normal text.

**Code change in `next-app/app/page.tsx`:**

```tsx
style={{
  background:
    "linear-gradient(160deg, rgba(20,120,120,0.88) 0%, rgba(31,58,95,0.88) 100%)",
}}
```

### Step 4 — A11Y-04: Alt text audit

**Audit scope:** All `<img>` tags in the `next-app/` directory.

**Findings from reading source files:**

| File | Element | Current alt | Status |
|------|---------|------------|--------|
| `components/Header.tsx` | Logo img | `"REACH Equity Lab"` | Pass — descriptive |
| `app/team/TeamGrid.tsx` | Member headshot | `` `Headshot of ${member.name}` `` | Pass — descriptive |
| `app/team/TeamGrid.tsx` | LinkedIn icon in overlay | `"LinkedIn"` | Marginal — acceptable since the parent `<a>` has `aria-label` with full context |
| `app/page.tsx` | (no img tags) | n/a | Pass |
| `components/Footer.tsx` | (verify in execution) | — | Check during execution |

**LinkedIn icon alt text improvement:** Since the parent `<a>` already provides the accessible name via `aria-label`, the nested `<img>` should use `alt=""` (empty string) to mark it as decorative and avoid redundant announcement by screen readers.

Change in `next-app/app/team/TeamGrid.tsx`:

```tsx
<img
  src="/icons/linkedin.svg"
  alt=""
  role="presentation"
  className="w-[54px] h-[54px]"
/>
```

**Footer:** During execution, read `next-app/components/Footer.tsx` and verify any images have descriptive alt text. Fix any missing or empty alt attributes on non-decorative images.
