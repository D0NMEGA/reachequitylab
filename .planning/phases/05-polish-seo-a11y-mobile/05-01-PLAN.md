# Plan 05-01: SEO Meta Tags — Descriptions, OG, and Twitter Cards

**Phase:** 5 — Polish: SEO, Accessibility & Mobile
**Wave:** 1
**Requirements:** SEO-01, SEO-02, SEO-03
**Status:** Ready

---

## Goal

Add unique `description`, Open Graph, and Twitter Card metadata to every Next.js route's `metadata` export. The global layout already defines a fallback title; each page-level export will extend or override it with page-specific values.

---

## Affected Files

| File | Change |
|------|--------|
| `next-app/app/layout.tsx` | Add site-wide OG defaults (og:image, og:type, twitter:card) to the root metadata export |
| `next-app/app/page.tsx` | Add `metadata` export with home-specific description, OG, and Twitter fields |
| `next-app/app/about/page.tsx` | Extend existing `metadata` with description, OG, and Twitter fields |
| `next-app/app/team/page.tsx` | Add `metadata` export (currently missing) with description, OG, and Twitter fields |
| `next-app/app/publications/page.tsx` | Extend existing `metadata` with description, OG, and Twitter fields |
| `next-app/app/events/page.tsx` | Extend existing `metadata` with description, OG, and Twitter fields |
| `next-app/app/connect/page.tsx` | Extend existing `metadata` with description, OG, and Twitter fields |

File: `next-app/app/layout.tsx`

Extend the existing `metadata` object to add site-wide Open Graph and Twitter Card defaults. These act as fallbacks for any page that doesn't override them.

```ts
export const metadata: Metadata = {
  title: "REACH Equity Lab",

    "REACH Equity Lab advances undergraduate research on racial, wage, and health disparities at UT Austin.",

```

### Step 4 — Add metadata export to Team page

File: `next-app/app/team/page.tsx`

`team/page.tsx` is a `"use client"` file and currently has no `metadata` export. In Next.js App Router, `metadata` cannot be exported from a client component. The fix is to split the page into a server wrapper that exports `metadata` and a client component for the interactive team grid.

**Approach:**
1. Create `next-app/app/team/TeamGrid.tsx` — move all existing client-side JSX (including `"use client"` directive, imports, `teamMembers` array, and `TeamPage` component body) into this new file. Export it as `TeamGrid`.
2. Rewrite `next-app/app/team/page.tsx` as a server component that exports `metadata` and renders `<TeamGrid />`.

New `next-app/app/team/page.tsx`:

```ts
import type { Metadata } from "next";
import { TeamGrid } from "./TeamGrid";

export const metadata: Metadata = {
  title: "Team | REACH Equity Lab",
  description:
    "Meet the undergraduate researchers behind REACH Equity Lab — founders, co-founders, and research team.",
  openGraph: {
    title: "Team | REACH Equity Lab",
    description:
      "Meet the undergraduate researchers behind REACH Equity Lab — founders, co-founders, and research team.",
    url: "https://reachequitylab.org/team",
    type: "website",
    images: [{ url: "/reach-banner.png" }],
  },
  twitter: {
    card: "summary_large_image",
    title: "Team | REACH Equity Lab",
    description:
      "Meet the undergraduate researchers behind REACH Equity Lab — founders, co-founders, and research team.",
    images: ["/reach-banner.png"],
  },
};

export default function TeamPage() {
  return <TeamGrid />;
}
```

---

## Notes

- The `url` values use the production domain `https://reachequitylab.org` — update if the canonical domain changes.
- Next.js merges page-level `metadata` with the root layout `metadata`, so OG image and twitter card defaults in `layout.tsx` serve as a safety net if a page omits them.
- The team page client/server split is necessary because Next.js App Router forbids `metadata` exports from `"use client"` modules.
